Основные тест-кейсы на основе пользовательских сценариев:

2 и 3. Логин и переадресация:
- логин в роли administrator и проверка переадресации на соответветствующую страницу.
- логин в роли contributor и проверка переадресации на соответветствующую страницу.
- логин в роли subscriber и проверка переадресации на соответветствующую страницу.
- негативный логин.

4. Для роли пользователя contributor или administrator - функциональность админки:
- заведение метакниги.
- заведение автора.
- заведение книги.
- удаление метакниги (если к ней ничего не привязано).
- удаление автора (если к нему ничего не привязано).
- удаление книги без глав (пустую книгу можно удалить, заполненную нельзя удалить).
- добавление текста в книгу (система считает, сколько глав разметил, проверка одинакового количества глав на разных языках).
- после добавления книги возможность просмотра добавленного текста.
- рефреш страницы.

6. Для любого пользователя (в т.ч. неавторизованного) - функциональность просмотра книг:
- посредством перехода по ссылкам (смена глав, смена языков, переход от книги к книге)
- должны работать сноски (должны быть под главой, относятся к книге)
- должны работать теги
- перемещение по ссылкам с помощью адресной строки (смена левой книги, правой книги, смена главы)

9. Выход из системы
- выход в роли administrator
- выход в роли contributor
- выход в роли subscriber
